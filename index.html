HTTP and Networks

  • Internet
    • Creation and concept: The internet is a big network that connects computers all over the world. Through the Internet, people can share information and communicate from anywhere with an Internet connection. It was created in the United States in the 1970s, but did not become visible to the general public until the 1990s.
    • Operation: The Internet works through a series of networks that connect devices around the world through telephone lines. Users are provided access to the Internet by Internet service providers. The widespread use of mobile broadband and Wi-Fi in the 21st century has allowed this connection to be wireless.
      • Reference: Comptia.org
    • Protocol
      • Concept: A network protocol is an established set of rules that determine how data is transmitted between different devices in the same network. Essentially, it allows connected devices to communicate with each other, regardless of any differences in their internal processes, structure or design.
      • Types:
        1. Transmission Control Protocol (TCP)
        2. Internet Protocol (IP)
        3. User Datagram Protocol (UDP)
        4. Post Office Protocol (POP)
        5. Simple Mail Transport Protocol (SMTP)
        6. File Transfer Protocol (FTP)
        7. Hyper Text Transfer Protocol (HTTP)
        8. Hyper Text Transfer Protocol Secure (HTTPS)
        9. Telnet
        10. Gopher
        • Reference: w3schools
      • RFC:
        RFC (stands for Request For Comments) is a document that describes the standards, protocols, and technologies of the Internet and TCP/IP.

    • HTTP Protocol
      • Concept: The Hypertext Transfer Protocol is an application protocol for distributed, collaborative, hypermedia information systems that allows users to communicate data on the World Wide Web.
      • Utility: HTTP was invented alongside HTML to create the first interactive, text-based web browser: the original World Wide Web. Today, the protocol remains one of the primary means of using the Internet.
      • Characteristics: Http is the request response protocol that allows web servers and browsers to exchange data over the web, uses the reliable TCP connections by default on TCP port 80 and is stateless means each request is considered as the new request. In other words, server doesn't recognize the user by default.

      • HTTP methods and their utility:
        GET requests a specific resource in its entirety HEAD requests a specific resource without the body content POST adds content, messages, or data to a new page under an existing web resource PUT directly modifies an existing web resource or creates a new URI if need be DELETE gets rid of a specified resource TRACE shows users any changes or additions made to a web resource OPTIONS shows users which HTTP methods are available for a specific URL CONNECT converts the request connection to a transparent TCP/IP tunne PATCH partially modifies a web resource

    • The process of loading a web page
      • A page load begins when a user selects a hyperlink, submits a form, or types a URL in a browser. This is also referred to as the initial request or the navigation start. The user's action sends a request across the network to the web application server. The request reaches the application for processing.
  • TCP
    • Concept:
      TTransmission Control Protocol (TCP) is a standard that defines how to establish and maintain a network conversation by which applications can exchange data.
    • Utility:
      TCP is used for organizing data in a way that ensures the secure transmission between the server and client. It guarantees the integrity of data sent over the network, regardless of the amount. For this reason, it is used to transmit data from other higher-level protocols that require all transmitted data to arrive.

  • IP
    • Concept:
      IP address stands for “Internet Protocol address.” The Internet Protocol is a set of rules for communication over the internet, such as sending mail, streaming video, or connecting to a website. An IP address identifies a network or device on the internet.
    • Utility:
      The purpose of an IP address is to handle the connection between devices that send and receive information across a network.
    • IP Adress
      The Internet and your network are linked together with Internet Protocol (IP) addresses. There are two kinds of IP addresses: static and dynamic.
    • Routing:
      Routing is the process of selecting a path for traffic in a network or between or across multiple networks. Broadly, routing is performed in many types of networks, including circuit-switched networks, such as the public switched telephone network (PSTN), and computer networks, such as the Internet.

  • Client-server applications vs P2P
    • Peer to peer and client server are two network types. In a client server network, multiple clients connect to a server. The clients require services, and the server provides required services. On the other hand, in a peer to peer network, there are no specific clients or servers.

    • MIME (Multipurpose Internet Mail Extensions) types tell browsers how to handle specific extensions. MIME types are often used to handle new technologies as they appear. When WAP technology first appeared, no one had these extensions set up on their server.

PROGRAM LENGUAGE

  • Program lenguage:
    • Compilated: Has to compilated and change to machine lenguage before it execution. (C++, JAVA, C)
    • Interpreted: Become to machine lenguage while be executed. (JavaScript, PHYTON)
  • Weakly Typed Languages vs Strongly Typed Languages:
    • Weakly Typed Languages: Weak typing is where we do not indicate the type of a variable when declaring it. The real difference is that we can assign, for example, an integer value to a variable that previously held a string.
      • Advantges:
        • We forgot to declare the type
        • We write less code
      • Disadvantages:
        • Insecurity: there is a possibility that an attacker discovers a vulnerability where we expect a variable of a certain type but another is received
        • You have to cast many times. Sometimes, we will have to force cast the variables so that they behave as we want and do not generate errors like the ones mentioned above.
    • Strongly Typed Languages: This is where we indicate the data type when declaring the variable. This type can never be changed. And we cannot operate between different types.
    • Advantges:
      • Expressive code: now we will know what type a function expects an argument
      • Less errors: We will forget to see the type of variable before doing operations with it
    • Disadvantages:
      • Write more code: we have to declare the type of variable when declaring it
  • What is the difference between static and dynamic typing?
    • A dynamic language (Lisp, Perl, Python, Ruby) is designed to optimize programmer efficiency, so you can implement functionality with less code. A static language (C, C++, etc) is designed to optimize hardware efficiency, so that the code you write executes as quickly as possible.
  • What is the timetomarket of a language?
    • It is the time that passes from when a language is released until an update arrives to replace it.
  • What is operator overloading?
    • It is one of the mechanisms that allows us to extend the capabilities of object-oriented programming languages.
  • What is string interpolation?
    • String interpolation is a technique that enables you to insert expression values into literal strings. It is also known as variable substitution, variable interpolation, or variable expansion. It is a process of evaluating string literals containing one or more placeholders that get replaced by corresponding values.
