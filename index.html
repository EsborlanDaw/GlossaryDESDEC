HTTP and Networks

  • Internet
    • Creation and concept: The internet is a big network that connects computers all over the world. Through the Internet, people can share information and communicate from anywhere with an Internet connection. It was created in the United States in the 1970s, but did not become visible to the general public until the 1990s.

    • Operation: The Internet works through a series of networks that connect devices around the world through telephone lines. Users are provided access to the Internet by Internet service providers. The widespread use of mobile broadband and Wi-Fi in the 21st century has allowed this connection to be wireless.

      • Reference: Comptia.org
    • Protocol
      • Concept: A network protocol is an established set of rules that determine how data is transmitted between different devices in the same network. Essentially, it allows connected devices to communicate with each other, regardless of any differences in their internal processes, structure or design.

      • Types:
        1. Transmission Control Protocol (TCP)
        2. Internet Protocol (IP)
        3. User Datagram Protocol (UDP)
        4. Post Office Protocol (POP)
        5. Simple Mail Transport Protocol (SMTP)
        6. File Transfer Protocol (FTP)
        7. Hyper Text Transfer Protocol (HTTP)
        8. Hyper Text Transfer Protocol Secure (HTTPS)
        9. Telnet
        10. Gopher

        • Reference: w3schools
      • RFC:
        RFC (stands for Request For Comments) is a document that describes the standards, protocols, and technologies of the Internet and TCP/IP.

    • ISO-OSI Levels
      • Levels
        1. Physical Layer
        2. Data Link Layer
        3. Network Layer
        4. Transport Layer
        5. Session Layer
        6. Presentation Layer
        7. Application Layer
        Reference: ISO

    • HTTP Protocol
      • Concept: The Hypertext Transfer Protocol is an application protocol for distributed, collaborative, hypermedia information systems that allows users to communicate data on the World Wide Web.

      • Utility: HTTP was invented alongside HTML to create the first interactive, text-based web browser: the original World Wide Web. Today, the protocol remains one of the primary means of using the Internet.

      • Characteristics: Http is the request response protocol that allows web servers and browsers to exchange data over the web, uses the reliable TCP connections by default on TCP port 80 and is stateless means each request is considered as the new request. In other words, server doesn't recognize the user by default.

      • Structure of an HTTP request and response packet:

      • HTTP methods and their utility:

        GET requests a specific resource in its entirety HEAD requests a specific resource without the body content POST adds content, messages, or data to a new page under an existing web resource PUT directly modifies an existing web resource or creates a new URI if need be DELETE gets rid of a specified resource TRACE shows users any changes or additions made to a web resource OPTIONS shows users which HTTP methods are available for a specific URL CONNECT converts the request connection to a transparent TCP/IP tunne PATCH partially modifies a web resource

        • Reference: Extrashop
    • The process of loading a web page
      • A page load begins when a user selects a hyperlink, submits a form, or types a URL in a browser. This is also referred to as the initial request or the navigation start. The user's action sends a request across the network to the web application server. The request reaches the application for processing.
